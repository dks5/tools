Полное руководство по веб-парсингу на Python

Этот документ содержит пошаговую инструкцию по созданию работающего веб-скрипта. Мы рассмотрим несколько подходов, решим самые распространенные проблемы и научимся сохранять данные в файл.

Часть 1: Подготовка рабочего окружения

1

Создание проекта через командную строку

Начнем с самого начала. Откройте терминал или командную строку на вашем компьютере. Мы создадим новую папку для проекта, а затем перейдем в нее.

Откройте командную строку (Terminal). В Windows это можно сделать, набрав `cmd` в поиске или нажав `Win + R` и введя `cmd`.

Перейдите на рабочий стол или в другую удобную для вас директорию с помощью команды `cd` (change directory).

cd Desktop

Создайте новую папку для проекта с именем `web_scraping`. Команда `mkdir` расшифровывается как "make directory" (создать директорию).

mkdir web_scraping

Теперь перейдите в созданную папку:

cd web_scraping
2

Создание и активация виртуального окружения

Виртуальное окружение **изолирует** ваш проект и его зависимости от других проектов на вашем компьютере. Это предотвращает конфликты версий и делает проект переносимым.

Windows
macOS / Linux

Создайте виртуальное окружение с помощью встроенного модуля `venv` (только для Windows):

python -m venv venv

Активируйте его, чтобы начать использовать:

.\venv\Scripts\activate

Создайте виртуальное окружение для macOS / Linux:

python3 -m venv venv

Активируйте его, чтобы начать использовать:

source venv/bin/activate
3

Установка необходимых библиотек

Для работы нам понадобятся две ключевые библиотеки:

  • requests: для отправки HTTP-запросов на сайт и получения HTML-кода.
  • beautifulsoup4: для парсинга (разбора) HTML-кода и поиска нужных элементов.

Установите их с помощью `pip` — менеджера пакетов Python. В терминале, где активировано виртуальное окружение, введите следующую команду:

pip install requests beautifulsoup4

Часть 2: Написание и запуск скрипта

4

Код для парсинга одной страницы

Вставьте этот код в файл `main.py` в вашей папке `web_scraping`. Он отправит HTTP-запрос на указанный URL, получит HTML-кода страницы и извлечет все элементы с классом "message-text".

# Импортируем необходимые библиотеки import requests from bs4 import BeautifulSoup # URL страницы для парсинга. Замените на нужный вам URL. url = 'https://semaglu.ru/forum/topic_756681/page_0' # Заголовки для имитации запроса от браузера. Это помогает избежать блокировки от некоторых сайтов. headers = {'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36'} try: # Отправляем GET-запрос на страницу response = requests.get(url, headers=headers) # Проверяем статус ответа. Если он не 200 (ОК), будет вызвано исключение. response.raise_for_status() # Создаем объект BeautifulSoup для парсинга HTML. # 'html.parser' - стандартный парсер, который идет с Python. soup = BeautifulSoup(response.text, 'html.parser') # Ищем все элементы с классом 'message-text'. Метод find_all() возвращает список. messages = soup.find_all(class_='message-text') if messages: # Если элементы найдены (список не пуст), проходим по ним в цикле for message in messages: # Извлекаем текст из элемента и удаляем лишние пробелы с помощью strip=True print(message.get_text(strip=True)) else: # Выводим сообщение, если элементы не были найдены на странице print("Элементы с классом 'message-text' не найдены.") except requests.exceptions.RequestException as e: # Обрабатываем возможные ошибки, связанные с запросом (например, нет сети, неверный URL) print(f"Произошла ошибка при получении страницы: {e}")

Чтобы запустить скрипт, находясь в папке проекта, введите в командной строке:

python main.py

Код для парсинга нескольких страниц

Чтобы парсить несколько страниц, вам нужно добавить цикл, который будет последовательно перебирать URL-адреса. Также, чтобы не нагружать сервер и избежать временной блокировки, мы добавим задержку между запросами с помощью `time.sleep()`. Кроме того, мы сохраним все собранные данные в файл `all_messages.txt`.

# Импортируем необходимые библиотеки: requests для запросов, BeautifulSoup для парсинга, time для задержек. import requests from bs4 import BeautifulSoup import time # Базовый URL, к которому будем добавлять номер страницы. base_url = 'https://semaglu.ru/forum/topic_756681/page_' # Список номеров страниц, которые мы хотим спарсить. pages = [0, 30, 60, 90] # Открываем файл для записи всех сообщений. 'a' означает режим добавления (append), # то есть новые данные будут добавляться в конец файла, а не перезаписывать его. # Указываем кодировку UTF-8 для корректной работы с русским языком. with open('all_messages.txt', 'a', encoding='utf-8') as f: # Запускаем цикл для перебора каждой страницы из нашего списка for page_number in pages: # Создаем полный URL для текущей страницы url = f"{base_url}{page_number}" print(f"Парсинг страницы: {url}") try: # Запрос страницы с имитацией браузера. headers = {'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36'} response = requests.get(url, headers=headers) response.raise_for_status() # Парсинг полученного HTML-кода soup = BeautifulSoup(response.text, 'html.parser') # Находим все сообщения messages = soup.find_all(class_='message-text') if messages: for message in messages: # Записываем текст сообщения в файл, добавляя перевод строки и разделитель f.write(message.get_text(strip=True) + '\n' + '---' + '\n') else: print(f"Элементы с классом 'message-text' не найдены на странице {url}") except requests.exceptions.RequestException as e: # Выводим подробное сообщение об ошибке, если запрос не удался print(f"Ошибка при получении страницы {url}: {e}") # Делаем задержку в 2 секунды перед следующим запросом, чтобы не перегрузить сервер. time.sleep(2) print("Парсинг завершен. Все сообщения сохранены в файл 'all_messages.txt'.")
